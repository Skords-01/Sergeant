# Hub: модулі на оку (рішення та ТЗ)

Цей документ фіксує **пріоритет доменів** для розвитку Hub і узгоджений з поточною архітектурою (`lazy`-модулі, `localStorage`, хаб у [`src/core/App.jsx`](../src/core/App.jsx)).

---

## 1. Пріоритет доменів (вибір)

| Пріоритет | Домен                 | Обґрунтування                                                                                                                                          |
| --------- | --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **1**     | **Здоров’я / рутина** | Уже є сильний стовп **Фізрук** (тренування, заміри, календар плану); логічно нарощувати звички, сон, легке харчування без роздування Фініка.           |
| **2**     | **Продуктивність**    | Нотатки, чеклісти, нагадування «крос-модульно» — після того, як дані користувача стабільно живуть в одному клієнті.                                    |
| **3**     | **Інфраструктура**    | Синхронізація між пристроями, обліковий запис, service worker для push — **не** окремі «вкладки хаба», а шар під Фініком/Фізруком/майбутніми модулями. |

**Висновок:** на найближчий горизонт розробки варто тримати в фокусі **здоров’я та рутину** (нові модулі або розширення Фізрука), а інфраструктуру планувати паралельно лише якщо з’являється реальна потреба в multi-device без ручного JSON.

## 2. Модуль «Рутина» (звички + Hub-календар)

Фактична реалізація: [`src/modules/routine/RoutineApp.jsx`](../src/modules/routine/RoutineApp.jsx), сховище [`hub_routine_v1`](../src/modules/routine/lib/routineStorage.js) (не окремий `hub_habits_v1`).

### 2.1 Мета

Трекер звичок із відмітками по днях, **streak**, інтеграція плану **Фізрука** і планових підписок **Фініка** в одному календарі, без сервера.

### 2.2 Дані (орієнтир)

- Один ключ `localStorage`: `hub_routine_v1` (звички, completions, теги, категорії, відтискання, нотатки до відміток, порядок звичок).
- Резервна копія: експорт/імпорт JSON з модуля (окремий формат `hub-routine-backup`).
- Дата — рядок **`YYYY-MM-DD`** у локальному календарі.

### 2.3 UI (реалізовано)

- Вкладки **Календар** / **Рутина** (налаштування, CRUD звичок, бекап).
- Зрізи часу (сьогодні, завтра, тиждень, місяць), тижневий ряд, місячна сітка, стрічка подій.
- Параметр URL `routineDay=YYYY-MM-DD` відкриває зріз одного дня.

### 2.4 Інтеграція з Hub

- Модуль підключено в [`App.jsx`](../src/core/App.jsx) як `module=routine`.
- Єдиний повний бекап Hub (якщо з’явиться) — окремий майбутній крок; поки експорт Рутини з модуля.

### 2.5 Не входить у поточну версію

- Спільний акаунт / синхронізація між пристроями.
- Push у фоні без обмежень браузера (можливий майбутній SW).
- Складна аналітика (лише streak + події в календарі).

### 2.6 Зв’язок з Фізруком

- План тренувань підтягується зі сховища Фізрука; перехід у модуль Фізрука зі стрічки.
